# 明远电子科技 ERP 管理系统 - 项目业务背景

## 一、项目概述

### 1.1 项目背景
本项目是为**深圳市明远电子科技有限公司**开发的企业资源计划（ERP）管理系统。该公司是一家专注于电子元器件研发、生产和销售的国家高新技术企业，主要产品包括贴片电容、贴片电阻、电感、IC芯片等电子元器件。

随着公司业务规模不断扩大，订单量持续增长，传统的Excel+纸质文档管理模式已无法满足业务需求，具体表现为：

- **信息孤岛**：采购、销售、库存数据分散，部门间信息不互通
- **库存失控**：热销型号缺货、滞销型号积压，库存周转率低
- **效率低下**：订单处理依赖人工记忆，经常出现遗漏或延误
- **决策困难**：管理层无法实时掌握企业经营状况

因此，公司决定引入ERP系统，实现采购、销售、库存、财务等核心业务的数字化管理。

### 1.2 项目目标
| 目标 | 预期效果 |
|------|----------|
| 业务数字化 | 建立统一数据平台，消除信息孤岛 |
| 流程规范化 | 规范采购、销售、库存业务流程 |
| 效率提升 | 提升运营效率30%以上 |
| 成本降低 | 降低库存成本20% |
| 数据支撑 | 为经营决策提供实时数据分析 |

### 1.3 项目范围
| 模块 | 功能说明 |
|------|----------|
| 用户管理 | 员工账号、角色权限、系统设置 |
| 供应商管理 | 供应商档案、价格管理、绩效评估 |
| 客户管理 | 客户档案、销售跟进、信用评级 |
| 商品管理 | 商品目录、价格策略、库存预警 |
| 采购管理 | 采购申请、订单审批、入库管理 |
| 销售管理 | 销售报价、订单处理、发货跟踪 |
| 库存管理 | 实时库存、出入库记录、盘点管理 |

- **IT能力参差不齐**：部分员工年龄较大，需考虑界面易用性

- **地域分散**：总部在深圳，异地仓库需要远程访问

## 三、核心业务流程

### 3.1 采购管理流程
```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 需求提出 │ → │ 采购申请 │ → │ 经理审批 │ → │ 采购下单 │ → │ 验收入库 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
                                              ↓
                                        ┌─────────┐
                                        │ 付款结算 │
                                        └─────────┘
```

**业务流程说明**：
1. **需求提出**：使用部门根据库存或需求提出采购申请
2. **采购申请**：采购员录入商品信息、供应商、价格、数量
3. **经理审批**：采购经理审核采购申请，批准或驳回
4. **采购下单**：审批通过后，向供应商正式下达采购订单
5. **验收入库**：供应商送货到货后，仓库验收并入库
6. **付款结算**：财务根据入库单据与供应商结算货款

**关键控制点**：
- 采购金额超过5000元需总经理审批
- 到货后3个工作日内完成验收
- 验收不合格可发起退换货流程

### 3.2 销售管理流程
```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 客户询价 │ → │ 报价确认 │ → │ 订单录入 │ → │ 库存确认 │ → │ 订单发货 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
      ↓                              ↓                              ↓
┌─────────┐                    ┌─────────┐                    ┌─────────┐
│ 建立档案 │                    │ 审批通过 │                    │ 收款确认 │
└─────────┘                    └─────────┘                    └─────────┘
```

**业务流程说明**：
1. **客户询价**：客户提出采购需求，销售员查询商品和价格
2. **报价确认**：根据采购量、合作关系等因素确定报价
3. **订单录入**：客户确认报价后，销售员创建销售订单
4. **库存确认**：系统检查库存是否充足，不足时提示备货
5. **订单发货**：仓库根据发货指令安排发货，更新物流信息
6. **收款确认**：财务确认收款后，订单完成闭环

**关键控制点**：
- 订单金额超过1万元需销售经理审批
- 库存不足时禁止超卖
- 发货后3天内录入物流单号

### 3.3 库存管理流程
```
┌──────────────────────────────────────────────────────────────┐
│                         库存管理                              │
├──────────────┬──────────────┬──────────────┬────────────────┤
│   入库操作    │   出库操作    │   库存预警    │    盘点管理     │
├──────────────┼──────────────┼──────────────┼────────────────┤
│ 采购入库     │ 销售出库      │ 安全库存预警  │ 月度盘点        │
│ 退货入库     │ 退货出库      │ 临期预警      │ 年度大盘        │
│ 调拨入库     │ 领用出库      │ 积压预警      │ 抽检盘点        │
│ 其他入库     │ 其他出库      │ 缺货预警      │ 不良报废        │
└──────────────┴──────────────┴──────────────┴────────────────┘
```

**库存计算公式**：
```
当前库存 = 期初库存 + 本期入库 - 本期出库
```

**库存预警规则**：
| 预警类型 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 安全库存预警 | 库存 ≤ 安全库存 | 推送采购建议 |
| 临期预警 | 保质期剩余 ≤ 30天 | 优先出库 |
| 积压预警 | 库存 > 库存上限 | 停止进货 |
| 缺货预警 | 库存 = 0 | 紧急补货 |

## 四、功能模块设计

### 4.1 供应商管理
**业务价值**：建立稳定的供应商渠道，降低采购成本

**核心功能**：
- 供应商档案：名称、联系人、资质、主营产品、合作条款
- 供应商分级：A级（战略）、B级（优质）、C级（合格）
- 价格管理：不同供应商的同产品比价分析
- 绩效评估：交货及时率、质量合格率、响应速度

**典型业务场景**：
> 采购员张三需要采购一批贴片电容，系统显示有3家合格供应商。通过供应商管理模块，可查看各家的历史报价、交货周期、合格率等数据，选择最优供应商下单。

### 4.2 客户管理
**业务价值**：深耕客户关系，提升客户满意度和复购率

**核心功能**：
- 客户档案：企业信息、联系人、信用等级、账期
- 客户分级：VIP客户、一般客户、潜在客户
- 跟进记录：每次沟通的时间、内容、跟进人
- 销售分析：客户贡献度、购买频次、产品偏好

**典型业务场景**：
> 销售员李四发现某VIP客户3个月没有下单。通过客户管理模块查看跟进记录，发现最后一次沟通是两个月前的质量问题。通过解决客户问题，该客户重新下单，季度销售额增加15万元。

### 4.3 商品管理
**业务价值**：建立完整商品目录，支持精准报价和库存管理

**核心功能**：
- 商品档案：SKU编码、名称、规格、单位、图片
- 价格管理：采购指导价、销售指导价、阶梯报价
- 库存设置：安全库存、库存上限、最小采购量
- 分类管理：按材质、规格、品牌等维度分类

**典型业务场景**：
> 销售员王五接到客户询价，需要快速报价。通过商品管理模块，输入客户采购量和期望交期，系统自动计算最优报价。报价单一键生成，响应时间从30分钟缩短到3分钟。

### 4.4 采购管理
**业务价值**：优化采购流程，降低采购成本，缩短采购周期

**核心功能**：
- 采购申请：商品、数量、供应商、期望价格、交期
- 审批流程：采购金额分段审批（经理→总监→总经理）
- 采购订单：正式下单、发货跟踪、到货验收
- 采购报表：采购金额统计、供应商对比、趋势分析

**典型业务场景**：
> 库存预警显示某型号IC库存不足。采购员发起采购申请，附上供应商比价表。采购经理在线审批，审批通过后自动生成采购订单。供应商发货后，系统提醒仓库准备验收。整个流程从发起申请到到货入库，由原来的5天缩短到3天。

### 4.5 销售管理
**业务价值**：提升订单处理效率，改善客户体验

**核心功能**：
- 销售报价：快速报价、报价有效期、历史报价查询
- 订单管理：订单录入、审批、变更、取消
- 发货管理：库存预占、发货指令、物流跟踪
- 销售报表：销售额、毛利、客户排名、产品排名

**典型业务场景**：
> 客户通过微信发送采购清单，共20个SKU。销售员将清单导入系统，1分钟生成报价单。客户确认后，点击一键下单。系统自动检查库存，发现2个SKU不足，提示可分批发货。客户同意后，订单提交审批。审批通过后，仓库立即安排有库存的商品发货。

### 4.6 库存管理
**业务价值**：实时掌握库存状况，降低库存成本，减少积压

**核心功能**：
- 实时库存：当前库存、在途库存、可用量
- 出入库记录：每笔出入库的来源、去向、经手人、时间
- 库存盘点：月度盘点、年度大盘、不良报废
- 库存分析：周转率、库龄分析、呆滞料预警

**典型业务场景**：
> 月末仓库主管进行库存盘点。通过系统导出库存明细表，与实物逐一核对。发现某批物料实际数量与系统不符，标记为盘点差异。完成盘点后，系统自动调整库存，并生成盘点报告。全程用时由原来的2天缩短到4小时。

## 六、技术架构

### 6.1 系统架构
```
┌─────────────────────────────────────────────────────────────┐
│                      客户端（Browser）                       │
│                 HTML5 + CSS3 + JavaScript                  │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP/HTTPS
┌──────────────────────────┴──────────────────────────────────┐
│                      应用服务器（Python Flask）                │
├────────────┬────────────┬────────────┬────────────┬────────────┤
│  用户模块   │ 供应商模块 │ 客户模块   │ 商品模块   │ 采购销售模块 │
├────────────┴────────────┴────────────┴────────────┴────────────┤
│                      业务逻辑层（Service）                    │
├─────────────────────────────────────────────────────────────┤
│                      数据访问层（SQLAlchemy）                │
└──────────────────────────┬──────────────────────────────────┘
                           │ SQL
┌──────────────────────────┴──────────────────────────────────┐
│                      数据库（SQLite）                        │
│                   用户、供应商、客户、商品、订单、库存          │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 接口设计
| 接口类型 | 说明 | 认证方式 |
|----------|------|----------|
| RESTful API | 前后端数据交互 | JWT Token |
| 内部接口 | 模块间调用 | 直接调用 |

### 6.3 安全设计
- 用户密码加密存储（PBKDF2）
- JWT Token 认证，有效期24小时
- 接口访问权限控制
- 操作日志全程记录
